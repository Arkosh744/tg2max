# tg2max

CLI-утилита для переноса истории чатов из Telegram в мессенджер Max.

## Как это работает

1. Экспортируешь чат из Telegram Desktop в JSON
2. Создаёшь бота в Max через @MasterBot
3. Запускаешь `tg2max` — он читает экспорт и отправляет сообщения в Max

## Что переносится

- Текстовые сообщения с форматированием (bold, italic, code, ссылки, цитаты)
- Фото и изображения
- Видео, аудио, документы
- Голосовые сообщения
- Имена авторов и время отправки
- Пересланные сообщения

## Установка

```bash
go install github.com/arkosh/tg2max/cmd/tg2max@latest
```

Или из исходников:

```bash
git clone https://github.com/arkosh/tg2max.git
cd tg2max
go build -o tg2max ./cmd/tg2max
```

## Подготовка

### 1. Экспорт чата из Telegram

1. Открой Telegram Desktop
2. Зайди в нужный чат
3. Нажми `...` → `Export Chat History`
4. Выбери формат **JSON** (Machine-readable JSON)
5. Отметь нужные типы данных (фото, видео, файлы)
6. Выбери папку для сохранения
7. Дождись завершения — появится файл `result.json`

### 2. Создание бота в Max

1. Найди `@MasterBot` в поиске Max
2. Отправь `/create`
3. Придумай username (>= 11 символов, заканчивается на `bot` или `_bot`)
4. Скопируй полученный токен
5. Добавь бота в нужный канал/чат как администратора

### 3. Конфигурация

Создай `config.yaml`:

```yaml
max_token: "токен-от-MasterBot"
rate_limit_rps: 25
cursor_file: "cursor.json"

mappings:
  - name: "Dev Chat"
    tg_export_path: "/path/to/export/result.json"
    max_chat_id: 123456789
```

`max_chat_id` — ID чата в Max. Его можно узнать, добавив бота в чат и отправив ему сообщение — в логах бота будет виден `chat_id`.

## Запуск

```bash
tg2max --config config.yaml
```

### Resume при сбое

При прерывании (Ctrl+C, ошибка сети) прогресс сохраняется в `cursor.json`. При повторном запуске миграция продолжится с места остановки.

### Формат сообщений в Max

Каждое сообщение отправляется в таком виде:

```
**Иван Петров** · 14 мар 2026, 15:42
Текст сообщения с **форматированием** и [ссылками](https://example.com)
```

## Ограничения

- Только JSON-экспорт из Telegram Desktop (не MTProto)
- Стикеры отправляются как фото
- Опросы, контактные карточки и реакции не переносятся
- Reply-цепочки не воспроизводятся (сообщения идут хронологически)
- Rate limit Max API: 30 RPS (утилита использует 25 по умолчанию)
